# LLM CACHE ROUTER

>Библиотека решает три связанные проблемы: дублирование LLM-запросов (→ семантический кэш), vendor lock-in (→ multi-provider роутинг) и непрозрачные расходы (→ cost tracker). Семантическое кэширование реально снижает затраты на 40–80%, а у LiteLLM есть баги с ложными совпадениями — это ваш шанс сделать лучше.
